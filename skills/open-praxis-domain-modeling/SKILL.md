# Alex Domain Modeling

Use this to make terms precise before implementation or writing.

## Workflow

1. Extract key terms from docs, code, or conversation.
2. Flag overloaded or ambiguous terms.
3. Propose canonical names and rejected alternatives.
4. Define relationships and lifecycle states.
5. Recommend where to store the glossary or ADR.

## Output

- Glossary.
- Ambiguities.
- Canonical decisions.
- Open questions.
