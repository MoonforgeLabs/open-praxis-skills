# Alex To Issues

Turn plans into small slices another agent can pick up.

## Issue Fields

- Title.
- Context.
- Scope.
- Acceptance criteria.
- Files or modules likely involved.
- Dependencies.
- Validation.
- Risk notes.

## Slicing Rules

- Prefer vertical user-visible slices.
- Avoid splitting all backend then all frontend.
- Keep each issue independently reviewable.
