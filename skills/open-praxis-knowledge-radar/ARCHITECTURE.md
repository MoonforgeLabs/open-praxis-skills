## 📁 模块说明

### 1. 核心功能 (CORE)

**新手必看**，完成快速开始后即可使用。

| 模块 | 功能 | 文件 |
|------|------|------|
| 知识捕获 | 手动输入、URL 抓取、钉钉订阅 | `radar.py` |
| 状态管理 | inbox → learning → learned → applied | `radar.py` |
| 学习编排 | 主题选择、笔记编写、应用出口 | `SKILL.md` |

**使用频率**: 每天

---

### 2. 增强功能 (ENHANCED)

**进阶用户**，需要额外配置。

| 模块 | 功能 | 文件 |
|------|------|------|
| 能力同步 | 检查 reference 更新 | `radar_enhanced.py` |
| 定期扫描 | 自动化同步 | `scheduled_sync.sh` |
| 钉钉通知 | 消息推送 | `dingtalk_notify.py` |

**使用频率**: 每周

---

### 3. 知识运维 (OPS)

**高级用户**，用于维护知识库质量。

| 模块 | 功能 | 文件 |
|------|------|------|
| 间隔重复 | FSRS-6 复习算法 | `knowledge_ops.py` |
| 知识 Lint | 质量检查和修复 | `knowledge_ops.py` |
| 内容分级 | A/B/C/D 分级 | `knowledge_ops.py` |
| 去重检查 | 重复条目检测 | `knowledge_ops.py` |
| 学习进度 | 完成度统计 | `knowledge_ops.py` |

**使用频率**: 每月

## 🎯 使用场景

### 场景 1: 新手入门（5 分钟）

**目标**: 快速上手，记录第一个想法

**步骤**:
1. 阅读 [QUICKSTART.md](QUICKSTART.md)
2. 运行 `python3 scripts/radar_enhanced.py status`
3. 添加第一个想法

**文档**: QUICKSTART.md

---

### 场景 2: 日常使用（每天）

**目标**: 记录和管理知识

**步骤**:
1. 记录新想法/URL
2. 开始学习
3. 记录学习笔记
4. 标记完成

**文档**: SKILL.md → 核心功能

---

### 场景 3: 定期维护（每周）

**目标**: 同步更新，发送报告

**步骤**:
1. 运行 `sync-all`
2. 检查更新
3. 发送钉钉通知

**文档**: SKILL.md → 增强功能

---

### 场景 4: 知识运维（每月）

**目标**: 维护知识库质量

**步骤**:
1. 运行 `knowledge_ops.py lint`
2. 处理重复条目
3. 复习间隔重复内容
4. 查看学习进度

**文档**: SKILL.md → 知识运维

## 💡 最佳实践

1. **从 inbox 开始**: 所有新条目都先进入 inbox
2. **定期同步**: 每周运行一次 `sync-all`
3. **使用触发词**: 在 Claude Code 中直接说触发词
4. **查看进度**: 使用 `progress` 命令查看完成度
5. **保持简洁**: 一次只学 1-3 个主题
