# 计费指南

## 概述

AI Usage Stats 使用 token 计费系统来估算 AI 模型的使用成本。费用基于输入、输出和缓存读取的 token 数量计算。

## 计费公式

```
费用 = (tokens_in × 输入费率 + tokens_out × 输出费率 + cache_read × 缓存费率) / 1,000,000
```

- **tokens_in**: 输入 token 数量
- **tokens_out**: 输出 token 数量
- **cache_read**: 缓存读取 token 数量
- **费率单位**: 美元/百万 tokens

## 模型名格式支持

系统支持多种模型名格式：

### 标准格式
```
claude-3-5-sonnet-20241022
gpt-4o-2024-05-13
```

### 代理服务格式
自动移除代理前缀（`pa/`、`proxy/`、`api/` 等）：
```
pa/claude-opus-4-6 → claude-opus-4-6
proxy/gpt-4o → gpt-4o
```

### 本地模型格式
```
ollama/llama3
mimo-v2.5-pro
```

### 版本号处理
自动移除版本后缀：
```
claude-3-5-sonnet-20241022 → claude-3-5-sonnet
gpt-4-v1 → gpt-4
model-latest → model
```

## 关键词匹配

| 关键词 | 匹配模型 |

## 计费示例

### 示例 1: Claude Opus 代理服务
```
模型: pa/claude-opus-4-6
输入 tokens: 1,000
输出 tokens: 500
缓存 tokens: 10,000

费用 = (1000 × 15.0 + 500 × 75.0 + 10000 × 1.5) / 1,000,000
     = (15,000 + 37,500 + 15,000) / 1,000,000
     = 67,500 / 1,000,000
     = $0.0675
```

### 示例 2: 本地 Ollama 模型
```
模型: ollama/llama3
输入 tokens: 2,000
输出 tokens: 800
缓存 tokens: 0

费用 = (2000 × 0.0 + 800 × 0.0 + 0 × 0.0) / 1,000,000
     = $0.00 (免费)
```

### 示例 3: GPT-4o Mini
```
模型: gpt-4o-mini
输入 tokens: 5,000
输出 tokens: 2,000
缓存 tokens: 0

费用 = (5000 × 0.15 + 2000 × 0.60 + 0 × 0.0) / 1,000,000
     = (750 + 1,200) / 1,000,000
     = 1,950 / 1,000,000
     = $0.00195
```

## 自定义费率

要添加或修改模型费率，编辑 `scripts/hook_collect.py` 中的 `MODEL_RATES` 字典：

```python
MODEL_RATES = {
    # 添加新模型
    "your-model-name": {"input": 1.0, "output": 2.0, "cache_read": 0.1},
    # ... 其他模型
}
```

## 注意事项

1. **费用是估算值**：基于公开的 API 定价，实际费用可能因折扣、套餐等有所不同
2. **本地模型免费**：Ollama、Llama 等本地模型的费用为 0
3. **缓存费率**：部分模型（如 Claude）支持缓存读取，费率较低
4. **未知模型**：使用默认费率，建议添加到 `MODEL_RATES` 中
5. **精度**：费用计算精确到小数点后 6 位

## 故障排除

### 费用显示为 0
- 检查模型名是否在 `MODEL_RATES` 中
- 确认 hook 正在工作：`python3 scripts/hook_install.py status`

### 费用异常高/低
- 检查模型名格式是否正确
- 确认是否使用了正确的费率（如 Opus vs Sonnet）

### 模型未识别
- 检查模型名是否包含特殊前缀（如 `pa/`）
- 尝试添加关键词匹配规则
